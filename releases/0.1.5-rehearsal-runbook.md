# Linmas 0.1.5 Release Rehearsal Runbook

Date: 2026-07-08

## Purpose

Validate the hardened CI/CD workflows, automated release notes, private-repo provenance skip logic, and documentation helpers introduced during the 0.1.4 hardening phase.

## Phase 1 — Local Preparation

- [ ] Checkout the `dev` branch.
- [ ] Ensure clean working tree: `git status --short`.
- [ ] Bump version in `package.json` and `package-lock.json` to `0.1.5`.
- [ ] Create `docs/releases/0.1.5.md` with release notes (title `Linmas 0.1.5`).
- [ ] Run the full local gate: `npm test && npm run validate && npm run pack:dry-run`.
- [ ] Verify the release notes matcher test passes: `node --test tests/github-workflows.test.mjs`.
- [ ] Sync local docs: `node scripts/sync-local-docs.mjs backup` (backup ignored docs before moving branches).
- [ ] Commit: `git add package.json package-lock.json docs/releases/0.1.5.md && git commit -m "chore: prepare 0.1.5 release"`.

## Phase 2 — Branch Alignment

- [ ] Merge into `main`: `git checkout main && git merge dev`.
- [ ] Push both branches: `git push origin dev && git push origin main`.
- [ ] Confirm CI runs pass on both `dev` and `main` (via GitHub Actions web UI).

## Phase 3 — Tag and Release

- [ ] Tag: `git tag v0.1.5 && git push origin v0.1.5`.
- [ ] Monitor the Release workflow on GitHub Actions.

## Phase 4 — Verification

- [ ] Confirm the Release workflow succeeds.
- [ ] Confirm the provenance job is skipped (green "skipped" in GitHub Actions UI).
- [ ] Verify GitHub Release body is populated: `gh release view v0.1.5 --json body`.
- [ ] Verify npm: `npm view linmas version dist-tags --json` should show `0.1.5` and `latest: 0.1.5`.
- [ ] Verify checksum script works on the published artifact: `node scripts/verify-checksum.mjs linmas-0.1.5.tgz`.

## Phase 5 — Cleanup

- [ ] Restore local ignored docs: `node scripts/sync-local-docs.mjs restore` (if working in a fresh checkout).
- [ ] Optionally remove or archive this runbook.
